【ディレクトリ・ファイル構造と実装ルール】
1. フォルダ構成の維持
- `cmd/`（実行フロー）と `internal/`（コアロジック: config, processor, restore 等）の構成を勝手に変更しないこと。
- ある程度の機能確認が可能になるまで、無闇にファイルを増やさず既存の構造の中で最適化すること。
- ただし、指示があればファイルやフォルダを増やしてもよい。
- 増やした場合、09_tree/09_tree.mdを更新すること。

2. ハードコードの厳禁
- システム制御に関する文字列やマジックナンバーは、ソースコード内に直接ハードコードしてはならない。
- 必ず設定ファイル（INI）の項目として扱うか、`internal/config/constants.go` 内の定数として定義し、一元管理すること。

3. リソースのハードリミット設定
- 低スペックPCでのフリーズを防ぐため、一定以上のメモリを消費した場合に自動で処理を制限したり、古いデータを圧縮・削除する安全装置をデフォルトで組み込む実装とすること。

4. コメントは日本語で書くこと

5. ログは日本語と英語で出力されること
