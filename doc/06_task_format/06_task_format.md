【AIへのタスク指示と出力フォーマット】
既存ソースのコードレビューおよび修正を行う際は、以下のルールに厳密に従うこと。
（※過去のセッションで学習した間違った修正や文脈はすべて忘れ、常にこのルールを起点とすること）

■ 問題の優先順位と対応方法
1. 致命的な問題（修正必須）: 具体的な修正コードを提示し、追記・改修を行うこと。
2. 修正推奨 / 軽微な問題: 概要を1〜2行で簡潔に指摘するに留め、詳細な解説やコードの提示は行わないこと。

■ 修正のアプローチ
- 保守費を減らすため、ただ場当たり的に直すのではなく、ファイルの切り分け（責務の分離）、リファクタリング、共通機能化を常に意識して修正を提案すること。

■ 修正内容の提示フォーマット（厳守）
修正を提案する際は、必ず以下のフォーマットで出力すること。（修正後だけを提示されても、どこを直せばいいか分からないため）

---
・対象ファイル名: `path/to/file.go`
・修正箇所の指摘: （なぜ直すのか、何を直すのかを簡潔に記載）

・修正前:
```go
// 修正前の該当コードのみ
