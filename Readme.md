## 🔥 なぜこれを作ったか

**「エンジニアとテスターを『スクショ刑務所』から解放したい」**

世の中には、最新のITツールやクラウドサービスが一切使えない「過酷な現場」が数多く存在します。
医療、教育、自治体、そして金融やレガシーな大企業。そこでは「最新技術の恩恵を受けること自体がリスクと見なされる、隔離された聖域（あるいは監獄）」です。

クラウドは「情報漏洩」と言われ、マクロ（VBA）は「ウイルス」と疑われ、Pythonを導入しようとすれば「許可に半年かかる」と言われる。その結果、もっともミスの許されない現場で、もっとも原始的な「手作業によるスクショ貼り付け」が、唯一の「安全な手段」として残ってしまいました。

**「ツールが使えないなら、環境の一部になればいい」**

`Evidence-Capture` は、理不尽な制約下で戦うすべての人のための「生存戦略」です。OSが提供するプリミティブなAPIのみを駆使し、外部依存を極限まで排除。どれほど過酷な環境であっても「ポン置き」で動作し、エンジニアが人間らしく働けるシェルターを構築します。

## 🛠 究極の「最悪環境」想定：動作保証の定義

一般的な「動作環境」の概念を捨て、ITの生存が困難な**極限状態**での完遂をターゲットに設計されています。

* **「石器時代」のネットワーク**: インターネット断絶（エアギャップ）はもちろん、社内LANすら接続不可、あるいは全ポート閉鎖済みのスタンドアロン端末。
* **「ガチガチ」の権限拘束**: 実行ファイルの拡張子変更すら制限され、一時フォルダ（Temp）への書き込みすら監視される標準ユーザー権限以下。
* **「枯れ果てた」リソース**: メモリ4GB以下、空き容量1GB未満。モダンなブラウザすら満足に動かない数世代前の低スペック端末。
* **「資産管理ソフト」の猛攻**: 外部からのOffice操作を「不審な挙動」として検知・遮断する、過剰に防衛的なセキュリティエージェントとの共存。

## 🚀 異常な現場に特化した「真」の機能

### 1. 現場の制約を無効化する「完全スタンドアロン」

* **ランタイム不要のネイティブバイナリ**: .NET FrameworkのバージョンやJavaの有無に左右されません。Goでビルドされた単一の `.exe` がOSの機能だけで完結して動作します。
* **レジストリ・環境変数への不干渉**: システムを一切汚しません。USBメモリから起動し、抜けば「何もなかった」ことになります。

### 2. Excelの「機嫌」を伺う高度な自動化制御

* **「COMの地獄」を回避**: 「セル入力中（編集モード）で操作を受け付けない」「ダイアログが開いている」といったExcel特有のフリーズ原因を精密に検知。安全に待機し、Excelが正気に戻ってから確実に処理を再開します。
* **リソースの自動クリーンアップ**: 数千枚の貼り付けでもメモリ消費を一定（100MB以下）に維持。古いPCでも動作を重くさせず、一時ファイルの残骸も自動で削除します。

### 3. 「撮れない」を許さない堅牢なフォールバック

* **多重のキャプチャ経路**: Win32 APIが阻害された場合、ビットブロック転送（BitBlt）やプリントスクリーン疑似命令に瞬時に切り替え、何が何でも証跡を残します。
* **「見えない裏側」も捕捉**: 画面外にはみ出たウィンドウや、他のウィンドウの裏に隠れている画面の完全キャプチャも可能です。
* **オートセーブ＆リカバリ**: 万が一の強制シャットダウン時も、直前までのデータを保護。作業の「巻き戻し」を最小限に抑えます。

### 4. 証跡としての「正当性」の証明

* **目次＆インデックスの自動生成**: 全シートへのリンク付き目次を自動生成。証跡のステータス（OK/NG）管理までをシームレスに行います。

## 💡 導入を検討されている方へ

あなたの現場が「日本で一番厳しい環境」だと思っているなら、ぜひこのツールを試してください。

`Evidence-Capture` は、OSやOfficeの気まぐれに耐え抜き、絶望的な単純作業に費やされていたあなたの「貴重な命（時間）」を取り戻します。

## 👨‍💻 開発者向け：ビルド方法（OSS版）

本リポジトリには、コアとなるキャプチャ・Excel制御ロジックが含まれています（商用版のライセンス認証・暗号化ロジックは除外されています）。
ご自身の環境でビルドして試す場合は、以下の手順を実行してください。

### 前提条件
- Go言語（1.20以上推奨）がインストールされていること。
- Windowsアイコン等のリソース埋め込みツール `rsrc` がインストールされていること。
  （未インストールの場合は `go install github.com/akavel/rsrc@latest` を実行）

### ビルド手順
同梱されている `build_oss.bat` をダブルクリックするか、コマンドプロンプトから以下のコマンドを実行してください。

```cmd
:: 1. manifestとiconリソースの生成
rsrc -manifest main.manifest -ico "icon\favicon.ico" -o cmd\capture\rsrc.syso

:: 2. 実行ファイルのビルド
go build -ldflags "-H windowsgui -s -w" -o Evidence-Capture-OSS.exe .\cmd\capture
